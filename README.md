## 搬瓦工安全性的几个维度，一个一个说

搬瓦工的安全性其实分几个层次。很多人问的是混在一起的，理清楚之后，你自己的顾虑在哪一块就会清晰很多。

### 平台硬件安全：企业级设备 + 全天候监控

搬瓦工官方明确说的是，全站使用企业级硬件，存储方案是 SSD RAID-10——也就是磁盘做了冗余，单块硬盘挂掉不会丢数据。

官网另一个表述是「工作人员 7×24 不间断监控所有服务和网络」，主动防范硬件或网络问题。每周还会对网络进行安全审计，官网本身有 SSL 证书保护。

这些不是什么特别稀奇的事，但对于长期建站或者跑业务的人来说，硬件稳定性是基础——连这个都没有的话，其他安全措施也是空谈。

### 数据备份：免费自动备份，这一点很多人不知道

搬瓦工给每台 VPS 提供免费的自动备份功能。实际用的时候，KiwiVM 控制面板里可以看到备份记录，支持一键从快照恢复。

这个细节比较重要。对于建站用户或者有重要数据在上面跑的人，自动备份是第一道防线。

### 网络传输安全：VPS 是工具，安全取决于你怎么配

这里要说一个常见的误解。

搬瓦工给你的是一台 KVM 架构的 VPS，网络传输是否加密，取决于你在上面跑什么服务、怎么配置的。如果你只是纯建站，启用 HTTPS 是你自己的事；如果你在上面自建代理服务，加密方式也是你的应用层决定的。

搬瓦工平台本身保障的是：你的 VPS 不受其他用户侵扰（KVM 独立虚拟化）、硬件层面的可靠性、以及账户访问的安全机制。

### 账户安全：2FA 是必须开的

讲真，KiwiVM 面板支持双重认证（2FA），但默认是不开的，需要你自己去设置。这个建议买了就第一时间开启，否则万一账户密码泄露，等于什么防线都没有。

另外一个要注意的是 SSH 密钥登录。默认 root 密码登录安全性比较低，建议在买完、第一次登录之后就切换成 SSH 密钥方式，同时顺手把 root 密码改成一个复杂度够高的。

### 账户被封和 IP 被封：两个完全不同的概念

这是很多人搞混的地方。

**账户被封**：触发条件是违反使用条款。搬瓦工明确禁止的行为包括发大量垃圾邮件、发动 DDoS 攻击、传播恶意软件、分发盗版内容等。正规使用——建站、跑脚本、部署应用、学 Linux——完全不在禁止范围内，账户没有风险。被警告三次才会被暂停，不是没有缓冲的。

**IP 被封（俗称 IP 被墙）**：这是 GFW 层面的问题，和搬瓦工本身的安全性关系不大。如果 IP 被封了，可以在 KiwiVM 里切换机房换一个 IP。

用过的人里被反馈最多的情况是：长期正规使用，IP 被墙的频率不高；但如果流量特征很明显，或者代理节点共享范围太广，被封的概率就会上升。这不是搬瓦工安全性的问题，是使用方式的问题。

**Q：用搬瓦工建站，数据安全吗？**
